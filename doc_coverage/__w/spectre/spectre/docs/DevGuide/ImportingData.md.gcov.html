       5             : # Importing data {#dev_guide_importing}
       6             : 
       7             : The `importers` namespace holds functionality for importing data into SpECTRE.
       8             : We currently support loading volume data files in the same format that is
       9             : written by the `observers`.
      10             : 
      11             : ## Importing volume data
      12             : 
      13             : The `importers::VolumeDataReader` parallel component is responsible for loading
      14             : volume data and distributing it to elements of one or multiple array parallel
      15             : components. As a first step, make sure you have added the
      16             : `importers::VolumeDataReader` to your `Metavariables::component_list`. Also make
      17             : sure you have a `Metavariables::Phase` in which you will perform registration
      18             : with the importer, and another for loading the data. Here's an example for such
      19             : `Metavariables`:
      20             : 
      21             : \snippet Test_VolumeDataReaderAlgorithm.hpp metavars
      22             : 
      23             : To register elements of your array parallel component for receiving volume data,
      24             : invoke the `importers::Actions::RegisterWithVolumeDataReader` action in the
      25             : `phase_dependent_action_list`. Here's an example:
      26             : 
      27             : \snippet Test_VolumeDataReaderAlgorithm.hpp register_action
      28             : 
      29             : Now you're all set to load a data file. To do so, invoke the
      30             : `importers::ThreadedActions::ReadVolumeData` action on the
      31             : `importers::VolumeDataReader`. A good place for this is the `execute_next_phase`
      32             : function of the array parallel component:
      33             : 
      34             : \snippet Test_VolumeDataReaderAlgorithm.hpp read_data_action
      35             : 
      36             : \snippet Test_VolumeDataReaderAlgorithm.hpp invoke_readvoldata
      37             : 
      38             : In the first snippet the `read_element_data_action` type alias is set to a
      39             : specialization of the `importers::ThreadedActions::ReadVolumeData` template.
      40             : Its first template parameter specifies an option group that determines the data
      41             : file to load, the second parameter is a typelist of the tags to import and fill
      42             : with the volume data, the third parameter is the action that is called upon
      43             : receiving the data on the array elements, and the last template parameter is the
      44             : parallel component on which the callback action will be invoked. See the
      45             : documentation of the `importers::ThreadedActions::ReadVolumeData` action for
      46             : details on these parameters. Here is more information on the parameters used in
      47             : the example above:
      48             : 
      49             : - The `VolumeDataOptions` in this example is an option group that supplies
      50             :   information such as the file name. You provide an option group that represents
      51             :   the data you want to import. For example, we have the
      52             :   `evolution::OptionTags::NumericInitialData` that represents numeric initial
      53             :   data for an evolution. In our example, we created a new class like this:
      54             : 
      55             :   \snippet Test_VolumeDataReaderAlgorithm.hpp option_group
      56             : 
      57             :   This results in a section in the input file that may look like this:
      58             : 
      59             :   \snippet Test_VolumeDataReaderAlgorithm2D.yaml importer_options
      60             : 
      61             : - The `::Actions::SetData` callback action in the example will be invoked to
      62             :   receive the data on each element. You can use this particular action to move
      63             :   the received data directly into the DataBox, but it's generally better to
      64             :   write a new action that does a few consistency checks on the data before
      65             :   moving it into the DataBox.
      66             : 
      67             : When the `importers::ThreadedActions::ReadVolumeData` is invoked, the data file
      68             : will be read and its data distributed to all elements of the array that have
      69             : been registered.
